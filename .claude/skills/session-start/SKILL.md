# Session Start — SPEC.md 컨텍스트 복원

세션 시작 시 SPEC.md에서 프로젝트 컨텍스트를 복원한다.

## Git 상태 확인

```bash
!`git log --oneline -10`
```

```bash
!`git status`
```

## 지시사항

1. **프로젝트 컨텍스트 읽기**:
   - `CLAUDE.md` 읽기 — 운영 규칙, 금지사항, 기술 스택, 설계 원칙 숙지
   - `SPEC.md` 전체를 읽되, 특히 다음 섹션을 집중 확인:
   - §5 Current Status — 현재 작업 상태 (가장 중요)
   - §6 Implementation Log — 구현 현황 및 완료 이력
   - §1~§4는 프로젝트 배경/설계/아키텍처/제약 파악용으로 읽기

2. **프로젝트 상태 요약 출력**:
   - 현재 진행 중인 작업
   - 최근 완료된 항목
   - 알려진 이슈/블로커
   - 다음 예정 작업

3. **인자가 제공된 경우**:
   - 사용자가 `$ARGUMENTS`로 오늘 작업할 내용을 전달하면, 해당 작업과 관련된 컨텍스트를 강조하여 출력
   - 관련 파일, 패턴, 주의사항 등을 SPEC.md에서 추출하여 제시

## 출력 형식

```
## 프로젝트 상태 요약

**현재 작업**: [§5에서 추출]
**최근 커밋**: [git log에서 추출]
**진행률**: [§6에서 완료/전체 비율]

### 알려진 이슈
- [있으면 나열]

### 오늘 작업 관련 컨텍스트
[인자가 있으면 관련 정보 출력]
```
