# ax-14-req-interview 검토 고도화 PRD

**버전:** v1
**날짜:** 2026-03-10
**작성자:** AX BD팀
**상태:** 🔄 검토 중

---

## 1. 요약 (Executive Summary)

**한 줄 정의:**
AX BD팀의 요구사항 인터뷰 스킬(ax-14-req-interview)의 외부 AI 검토 파이프라인을 고도화하여, 수동 복사-붙여넣기 방식에서 API 자동 호출 + 모델 선택 가능한 구조로 전환한다.

**배경:**
현재 ax-14-req-interview 스킬은 Phase 2(검토 패키지 생성)에서 4개 AI(ChatGPT, Gemini, Claude, Grok)별 프롬프트 파일을 생성하지만, 사용자가 각 AI 서비스에 직접 복사-붙여넣기 해야 한다. 또한 검토에 사용할 모델이 고정되어 있어 유연성이 부족하다.

**목표:**
- 외부 AI 검토를 API 호출로 자동화
- 검토 모델을 사용자가 선택/변경 가능하게
- 검토 프롬프트 품질 개선 및 역할 세분화
- 검토 결과를 자동 수집하고 구조화된 포맷으로 파싱

---

## 2. 문제 정의

### 2.1 현재 상태 (As-Is)
- 검토 프롬프트가 마크다운 파일로만 생성됨 → 사용자가 4개 AI에 각각 수동 복붙
- 검토 모델이 ChatGPT/Gemini/Claude/Grok 4개로 고정됨
- 검토 결과도 사용자가 수동으로 붙여넣거나 파일로 저장해야 함
- 각 AI별 프롬프트가 범용적이어서 모델별 특성을 충분히 활용하지 못함
- 1회 검토 라운드에 최소 20~30분의 수동 작업 필요

### 2.2 목표 상태 (To-Be)
- 스킬 내에서 선택된 AI 모델의 API를 직접 호출하여 검토 자동 수행
- 사용자가 검토에 사용할 모델을 설정/변경 가능
- 검토 결과가 자동으로 수집되어 구조화된 피드백 문서로 생성
- 모델별 특화 프롬프트로 검토 품질 향상
- 1회 검토 라운드가 5분 이내로 단축

### 2.3 시급성
- ax-14-req-interview 스킬이 S349에서 막 설치됨 → 초기 사용 단계에서 고도화하면 팀 도입 가속
- 현재 수동 워크플로우는 팀원이 사용하기에 진입 장벽이 높음
- MCP 서버화의 전제 조건 → 스킬이 먼저 안정화되어야 MCP로 전환 가능

## 5. 성공 기준

### 5.1 정량 지표 (KPI)
| 지표 | 현재값 | 목표값 | 측정 방법 |
|------|--------|--------|-----------|
| 1회 검토 라운드 소요 시간 | ~25분 (수동) | ~5분 (자동) | 타임스탬프 비교 |
| 검토 완료율 (4개 AI 모두) | ~50% (수동이라 일부만) | 90%+ | feedback.md 생성 여부 |
| 팀원 사용률 | 1명 (개발자만) | 3명+ | 실제 사용 건수 |

### 5.2 MVP 최소 기준
- 최소 2개 AI 모델의 API 자동 호출이 동작
- 검토 결과가 자동으로 feedback.md에 구조화되어 저장
- 팀원 2명 이상이 사용 후 "쓸만하다" 피드백

### 5.3 실패/중단 조건
- API 비용이 1회 검토당 $1 이상 발생하여 지속 불가
- 팀원이 수동 복붙이 더 낫다고 판단
- API 호출 안정성이 70% 미만

## 6. 제약 조건

### 6.1 일정
- 목표 완료일: 미정 (여유롭게 진행)
- 마일스톤: Phase 1(모델 레지스트리 + API 호출) → Phase 2(프롬프트 + 파싱) → Phase 3(비용 + 폴백)

### 6.2 기술 스택
- 런타임: Claude Code bash (node/curl)
- API 클라이언트: node fetch 또는 curl
- 설정: YAML/JSON 설정 파일 (모델 레지스트리)
- API 키: 환경변수

### 6.3 인력/예산
- 투입 가능 인원: 1명 (Sinclair)
- API 비용: 제한적 (건당 비용 최소화 필요)

### 6.4 컴플라이언스
- API 키 노출 금지
- PRD에 사내 민감 정보 포함 시 외부 전송 주의
