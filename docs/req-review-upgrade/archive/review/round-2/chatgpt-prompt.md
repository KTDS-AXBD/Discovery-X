## 1. 요약 (Executive Summary)

**한 줄 정의:**
AX BD팀의 요구사항 인터뷰 스킬(ax-14-req-interview)의 외부 AI 검토 파이프라인을 고도화하여, 수동 복사-붙여넣기 방식에서 API 자동 호출 + 모델 선택 가능한 구조로 전환한다.

**배경:**
현재 ax-14-req-interview 스킬은 Phase 2(검토 패키지 생성)에서 4개 AI(ChatGPT, Gemini, Claude, Grok)별 프롬프트 파일을 생성하지만, 사용자가 각 AI 서비스에 직접 복사-붙여넣기 해야 한다. 또한 검토에 사용할 모델이 고정되어 있어 유연성이 부족하다.

**목표:**
- 외부 AI 검토를 API 호출로 자동화
- 검토 모델을 사용자가 선택/변경 가능하게
- 검토 프롬프트 품질 개선 및 역할 세분화
- 검토 결과를 자동 수집하고 구조화된 포맷으로 파싱

**프로토타입 검증 (S352):**
- review-api.mjs v2 프로토타입 구현 완료
- ChatGPT + Gemini 병렬 호출: 28.3초에 2개 모델 검토 완료
- 착수 판단 파싱 성공률: 100% (2/2)
- 비용: ~$0.013/라운드 (2개 모델 기준)

## 2. 문제 정의

### 2.1 현재 상태 (As-Is)
- 검토 프롬프트가 마크다운 파일로만 생성됨 → 사용자가 4개 AI에 각각 수동 복붙
- 검토 모델이 ChatGPT/Gemini/Claude/Grok 4개로 고정됨
- 검토 결과도 사용자가 수동으로 붙여넣거나 파일로 저장해야 함
- 각 AI별 프롬프트가 범용적이어서 모델별 특성을 충분히 활용하지 못함
- 1회 검토 라운드에 최소 20~30분의 수동 작업 필요

### 2.2 목표 상태 (To-Be)
- 스킬 내에서 선택된 AI 모델의 API를 직접 호출하여 검토 자동 수행
- 사용자가 검토에 사용할 모델을 설정/변경 가능
- 검토 결과가 자동으로 수집되어 구조화된 피드백 문서로 생성
- 모델별 특화 프롬프트로 검토 품질 향상
- 1회 검토 라운드가 5분 이내로 단축 (API 호출 30초 + 결과 확인 4분 30초)

### 2.3 시급성
- ax-14-req-interview 스킬이 S349에서 설치됨 → 초기 사용 단계에서 고도화하면 팀 도입 가속
- 현재 수동 워크플로우는 팀원이 사용하기에 진입 장벽이 높음
- MCP 서버화의 전제 조건 → 스킬이 먼저 안정화되어야 MCP로 전환 가능
- S352에서 프로토타입(review-api.mjs v2) 검증 완료 → 즉시 스킬 통합 가능
