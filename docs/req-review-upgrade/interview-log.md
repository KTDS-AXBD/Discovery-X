## Part 1: 왜 (목적/문제)

**Q: 어떤 문제가 있어서, 또는 어떤 기회를 잡으려고 이걸 만들려고 하시나요?**

A: 지금 만든 인터뷰를 통한 요구사항 구체화하는 스킬을 실제로 사용하고 발전시켜서 MCP로 만들려고 해. 그리고, 타 LLM에게 검토하는 부분들에 대해서 구체적으로 어떤 모델을 사용하는지 점검하고 필요하다면 변경/선택 기능을 넣고 싶어.

**Q: 시급성은?**

A: 스킬 고도화가 우선. MCP화는 다음 단계.

**정리:**
- 현재 스킬의 외부 AI 검토 부분이 수동 복사-붙여넣기 방식
- 모델이 고정(ChatGPT, Gemini, Claude, Grok)되어 유연성 부족
- 우선 스킬 자체 품질을 높이고, 추후 MCP 서버로 발전

---

## Part 2: 누구를 위해 (사용자/이해관계자)

**Q: 이걸 누가 사용하게 되나요?**

A: AX BD팀

**Q: 기술 수준은?**

A: 혼합 — Claude Code CLI + Claude Desktop 사용자 모두

**정리:**
- 주 사용자: AX BD팀원 (Sinclair 포함)
- 사용 환경: CLI(Claude Code) + Desktop(Claude Desktop) 혼합
- MCP 서버화 시 두 환경 모두 커버 가능

---

## Part 3: 무엇을 (범위/기능)

**Q: 핵심 기능을 딱 한 문장으로?**

A: 검토 고도화만 — API 자동 호출 + 프롬프트 개선 둘 다.

**Q: Out-of-scope은?**

A: MCP 패키징은 이번 범위에서 제외.

**정리:**
- Must Have: 모델 선택/변경 기능 + API 자동 호출 + 프롬프트 개선 + 결과 자동 수집/파싱
- Out-of-scope: MCP 서버 패키징 (다음 단계), 인터뷰 파트(Part 1~5) 변경 없음

---

## Part 4: 어떻게 판단할 것인가 (성공 기준)

**Q: 성공 기준은?**

A: 사용성 — 팀원 피드백 기반.

**Q: MVP 기준은?**

A: 팀원 2명 이상이 써보고 긍정 피드백.

**정리:**
- 성공 기준: 팀원이 실제로 사용하고 "쓸만하다" 피드백
- MVP: 팀원 2명+ 사용 후 긍정 피드백
- 실패 조건: 팀원이 직접 프롬프트 복붙이 더 낫다고 판단

---

## Part 5: 제약과 리소스 (현실 조건)

**Q: 제약 조건은?**

A: API 비용 제한 — 여러 API(OpenAI, Google AI 등) 사용 가능하나 비용 제한적.

**정리:**
- API 비용: 제한적이나 여러 API 사용 가능 (OpenAI, Google AI, xAI 등)
- 기술: Claude Code bash에서 API 호출 가능 (curl/node)
- 일정: 별도 제한 없음
- 보안: API 키 관리 필요 (.dev.vars 또는 환경변수)
