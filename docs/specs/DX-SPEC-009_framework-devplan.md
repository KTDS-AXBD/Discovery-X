# KT DS AX Business Development Framework

## Discovery X Porting Development Plan

**작성일:** 2026-02-18\
**작성자:** AX BD Team

------------------------------------------------------------------------

# 1. 목적

본 문서는 KT DS의 신규 AX 사업 개발을 위한 **산업 × 기능 매트릭스 기반
프레임워크**를 정의하고,\
이를 현재 개발 중인 **Discovery X 시스템에 포팅하기 위한 개발
계획서**이다.

목표는 다음과 같다:

-   산업(X축) × 기능(Y축) 매트릭스 기반 신규 사업 발굴 체계 수립
-   3개월 / 1\~2년 / 3년 이내의 시계열 전략 관리
-   C-Level 관점 + 실무자 관점 통합 의사결정 구조 구축
-   Discovery X 내에서 데이터 기반 우선순위 관리 및 자동화 연계

------------------------------------------------------------------------

# 2. 프레임워크 구조

## 2.1 X축: 산업군 (Broad Coverage)

전체 산업군을 포괄하되, KT DS 강점 산업군에 가중치 적용

예시:

-   금융
-   공공
-   통신
-   제조
-   유통/커머스
-   헬스케어
-   에너지
-   기타 전략 신산업

## 2.2 Y축: 기능(Function) 재정렬

SAP 기능 기반 + AI 서비스 중심 재정렬

-   재무/회계
-   공급망/물류
-   HR
-   CRM/고객경험
-   운영/생산
-   데이터/분석
-   AI 자동화
-   AI 기반 프로세스 혁신
-   AI 서비스 플랫폼 구축

------------------------------------------------------------------------

# 3. 시간 관점 (Time Dimension)

  구분   기간       전략적 의미
  ------ ---------- ----------------------------------
  단기   0\~3개월   즉시 실행 가능 / 빠른 매출 기회
  중기   1\~2년     준비형 전략 / 조직 역량 축적
  장기   3년 이내   미래 산업 대응 / 선제적 포지셔닝

Discovery X 내에서 각 매트릭스 항목은 반드시 Time Tag를 보유한다.

------------------------------------------------------------------------

# 4. 의사결정 관점 통합

## 4.1 C-Level 관점

-   전략 적합성
-   수익성
-   시장 확장성
-   브랜드 영향력
-   투자 대비 기대 수익

## 4.2 실무자 관점

-   실행 가능성
-   기술 난이도
-   기존 레퍼런스 보유 여부
-   인력 가용성
-   리스크 요소

각 매트릭스 셀은 C-Level Score + Execution Score를 동시에 가진다.

------------------------------------------------------------------------

# 5. Discovery X 포팅 설계

## 5.1 데이터 모델 확장

신규 Entity 추가:

-   Industry
-   Function
-   Time Horizon
-   C-Level Score
-   Execution Score
-   Revenue Potential
-   Strategic Weight

기존 Stage-Gate 구조와 연결:

S0 → S1 → S2 → S3 → S4

## 5.2 UI 설계 방향

1.  Matrix View (Heatmap)
2.  Time Filter View
3.  Executive Dashboard
4.  Operational Dashboard

------------------------------------------------------------------------

# 6. 개발 로드맵

## Phase 1 (2주)

-   DB Schema 확장
-   Matrix 기본 구조 구현
-   Time Dimension 필드 추가

## Phase 2 (2\~4주)

-   Scoring Logic 구현
-   Heatmap UI 구현
-   Executive Summary Dashboard 구축

## Phase 3 (4\~8주)

-   자동 우선순위 추천 엔진
-   수익 추정 모델 연동
-   Action Play 연결

------------------------------------------------------------------------

# 7. 성공 기준 (KPI)

-   산업 × 기능 조합 최소 30개 이상 등록
-   3개월 내 실행 과제 명확화
-   C-Level 보고용 대시보드 운영 개시
-   Discovery X 내 우선순위 기반 Gate 의사결정 자동화

------------------------------------------------------------------------

# 8. 결론

본 프레임워크는 단순한 아이디어 정리가 아니라, KT DS의 전략적 신규 AX
사업 개발을 위한 **전사 의사결정 시스템**이다.

Discovery X는 이를 실행하는 엔진이며, 본 개발 계획은 그 엔진에 전략적
사고 체계를 이식하기 위한 로드맵이다.
