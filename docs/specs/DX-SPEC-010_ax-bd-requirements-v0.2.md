# 개발 티켓 단위 요구사항 (v0.2)

본 문서는 PRD v0.1을 기반으로 PoC 개발 착수를 위해 요구사항을 개발 티켓 단위로 분해한 문서입니다.

---

## EPIC 1. 시장 소스 수집 & 요약

### T1-1. 관심 주제 기반 소스 수집 설정
- 사용자 관심 키워드/태그 등록
- RSS, URL 선택 입력
- 사용자별 설정 분리

### T1-2. 소스 리스트 저장 및 상태 관리
- 상태: New / Viewed / Archived
- 시간순 정렬

### T1-3. 소스 클릭 시 즉시 요약 생성
- 한 줄 요약
- 핵심 포인트 3~5개
- 원문 링크 포함

---

## EPIC 2. 개인 Workspace 탐색

### T2-1. 소스 기반 대화 시작
- 요약 후 즉시 대화 가능
- 소스 컨텍스트 유지

### T2-2. 연관 소스 추천
- 기술 / 서비스 / 시장 사례 기반 추천
- 최소 3개 이상

### T2-3. Workspace 히스토리 관리
- 대화 및 소스 히스토리 유지
- 재진입 가능

---

## EPIC 3. 아이디어 후보 생성

### T3-1. 아이디어 후보 자동 생성
- 최대 3개 후보 생성
- 제목 + 요약 제공

### T3-2. 아이디어 후보 선택
- 1개 선택
- 아이디어 상태로 승격

---

## EPIC 4. 아이디어 공통 템플릿

### T4-1. 아이디어 템플릿 자동 채움
필수 항목:
1. 가설
2. 근거 (링크 포함)
3. 타겟
4. 가치 제안

### T4-2. 아이디어 템플릿 수동 수정
- 모든 필드 편집 가능
- 수정 내용 저장

---

## EPIC 5. 팀 공유 & 논의

### T5-1. 아이디어 팀 공유
- 개인 → 팀 공간 공유

### T5-2. 팀 아이디어 리스트 뷰
- 카드/리스트 형태
- 상세 템플릿 접근

---

## EPIC 6. 기본 UI 구조

### T6-1. 메인 작업 화면 (3-Pane)
- 좌: 소스/히스토리
- 중: 대화
- 우: 요약/아이디어

### T6-2. 팀 논의 화면
- 팀 공유 아이디어 전용

---

## EPIC 7. 기술/운영 최소 요건

### T7-1. LLM 모델 선택 구조
- 개인 API Key 사용
- 모델 선택 가능

### T7-2. 로그 & 오류 처리
- 요약/생성 실패 시 재시도
- 에러 메시지 표시
