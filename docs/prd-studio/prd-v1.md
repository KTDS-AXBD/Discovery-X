## 1. 요약 (Executive Summary)

**한 줄 정의:**
Discovery-X 앱 내에서 대화형 인터뷰 → PRD 자동 생성 → AI 다중 검토 → 피드백 반영 → 착수 판단까지 수행하는 웹 UI 기능.

**배경:**
현재 요구사항 기획은 Claude Code CLI 전용 스킬(`ax-14-req-interview`)로만 가능하다. 비개발자 팀원은 CLI에 접근할 수 없어 기획 프로세스에 참여하지 못하고, 개발자도 CLI와 웹 앱 사이를 오가며 비효율적으로 작업한다. Discovery-X의 아이디어/사업제안 흐름과 요구사항 기획이 완전히 분리되어 있어 컨텍스트 전환 비용이 높다.

**목표:**
- 비개발자를 포함한 팀 전체가 웹 UI에서 PRD 기획/검토 가능
- 아이디어 → PRD → 사업제안 흐름이 하나의 앱에서 완결
- CLI 스킬의 검토 품질(다중 AI 검토, 스코어카드)을 웹에서도 동일하게 제공

## 2. 문제 정의

### 2.1 현재 상태 (As-Is)
- 인터뷰 스킬은 Claude Code CLI(`/ax-14-req-interview`) 전용
- 비개발자 팀원(5명 중 4명)은 CLI 환경에 접근 불가
- PRD 검토는 `review-api.mjs` Node 스크립트를 로컬에서 실행해야 함
- 아이디어 분석 결과와 PRD 기획이 별개 도구에서 관리됨
- 검토 결과(스코어카드, 피드백)가 로컬 파일로만 존재

### 2.2 목표 상태 (To-Be)
- 웹 UI에서 대화형 인터뷰 진행 → PRD 자동 생성
- 생성된 PRD에 대해 AI 다중 검토를 앱 내에서 실행
- 검토 피드백 → PRD 수정 → 재검토 사이클을 반복
- 스코어카드 자동 채점 → 착수 판단 지원
- 아이디어 분석 데이터를 PRD 초안에 자동 반영
- 완성된 PRD를 사업제안에서 참조

### 2.3 시급성
- 운영 실험(2026-01-31~) 진행 중이나 비개발자 기획 참여 수단 부재
- 수동 기획 과정에서 맥락 손실 및 반복 작업 발생
- 이 기능이 없으면 인터뷰 스킬의 사용자 범위가 개발자 1명으로 제한

## 6. 제약 조건

### 6.1 일정
- 목표 완료일: [미정] — 운영 실험 기간 내 MVP
- 마일스톤: Phase 1 (DB+API) → Phase 2 (인터뷰 UI) → Phase 3 (검토+스코어카드)

### 6.2 기술 스택
- 프론트엔드: React 19 + Tailwind CSS 4 + @axis-ds
- 백엔드: Remix v2 (Cloudflare Pages Edge Runtime)
- DB: D1 (SQLite) + Drizzle ORM
- AI 호출: Edge Runtime에서 직접 `fetch` (Node child_process 불가)
- 기존 패턴: SSE 스트리밍 (api.chat.ts, api.ideas.$id.analyze.ts 참조)

### 6.3 인력/예산
- 투입 가능 인원: 1명 (Sinclair)
- AI API 예산: PRD당 $1 이하 (월 $30 미만 목표)

### 6.4 컴플라이언스
- PRD 내용은 외부 AI API로 전송됨 — 민감 정보 포함 시 주의 필요
- API 키 관리: 서버사이드에서만 사용 (클라이언트 노출 없음)
- 기존 Discovery-X 인증 체계(Google OAuth + 세션) 준수
