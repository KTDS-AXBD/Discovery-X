## 1. 요약 (Executive Summary)

**한 줄 정의:**
Discovery-X 앱 내에서 대화형 인터뷰 → PRD 자동 생성 → AI 다중 검토 → 피드백 반영 → 착수 판단까지 수행하는 웹 UI 기능.

**배경:**
현재 요구사항 기획은 Claude Code CLI 전용 스킬(`ax-14-req-interview`)로만 가능하다. 비개발자 팀원은 CLI에 접근할 수 없어 기획 프로세스에 참여하지 못하고, 개발자도 CLI와 웹 앱 사이를 오가며 비효율적으로 작업한다.

**목표:**
- 비개발자를 포함한 팀 전체가 웹 UI에서 PRD 기획/검토 가능
- 아이디어 → PRD → 사업제안 흐름이 하나의 앱에서 완결
- CLI 스킬의 검토 품질(다중 AI 검토, 스코어카드)을 웹에서도 동일하게 제공

## 2. 문제 정의

### 2.1 현재 상태 (As-Is)
- 인터뷰 스킬은 Claude Code CLI 전용 — 비개발자(5명 중 4명) 접근 불가
- PRD 검토는 로컬 Node 스크립트 실행 필요
- 아이디어 분석 결과와 PRD 기획이 별개 도구에서 관리
- 검토 결과(스코어카드, 피드백)가 로컬 파일로만 존재

### 2.2 목표 상태 (To-Be)
- 웹 UI에서 대화형 인터뷰 → PRD 자동 생성 → AI 검토 → 피드백 반영 → 착수 판단
- 아이디어 데이터를 PRD 초안에 자동 반영, 완성 PRD를 사업제안에서 참조

### 2.3 시급성
- 운영 실험 진행 중이나 비개발자 기획 참여 수단 부재
- 이 기능이 없으면 인터뷰 스킬 사용자가 개발자 1명으로 제한

## 3. 사용자 및 이해관계자

### 3.1 주 사용자
| 구분 | 설명 | 주요 니즈 |
|------|------|-----------|
| BD팀 비개발자 | 사업 기획/분석 담당 | CLI 없이 웹에서 PRD 기획 수행 |
| BD팀 개발자 | Discovery-X 개발/운영 | 아이디어→PRD→제안 흐름을 앱 내에서 완결 |

### 3.2 사용 환경
- 기기: PC (데스크톱 웹) / 네트워크: 인터넷 / 기술 수준: AI 챗봇 익숙

### 3.3 핵심 사용자 여정

**여정 A: 아이디어 → PRD (비개발자)**
```
아이디어 상세 → "PRD 작성" → 인터뷰 화면 (분석 데이터 사전 로드)
→ 채팅형 5파트 인터뷰 (파트별 1질문씩, 진행률 표시)
→ 인터뷰 완료 → 요약 확인 → PRD v1 자동 생성
→ PRD 미리보기/편집 → "AI 검토" 버튼 → 검토 진행 로딩
→ 피드백 뷰어 (AI별 탭) + 스코어카드 → 수정 → 재검토 또는 착수
```

**여정 B: 독립 PRD 작성**
```
Lab > PRD Studio 탭 → "새 PRD" → 프로젝트명 → 인터뷰 → (A와 동일)
```

### 3.4 비개발자 UX 설계 방향

- **가이드 기반 인터뷰**: 질문마다 예시 답변을 접힘 형태로 제공 ("예: '현재 수동으로 엑셀을 관리하는데 30분씩 걸려요'")
- **진행률 표시**: 5파트 프로그레스 바 + 현재 파트명 표시 → 얼마나 남았는지 인지
- **중간 저장**: 브라우저 닫아도 인터뷰 진행 상태 DB 저장 → 재진입 시 이어하기
- **결과 즉시 확인**: PRD 생성 후 마크다운 미리보기 + "이해가 안 되는 부분이 있나요?" CTA
- **검토 진행 상태**: 모델별 진행/완료 뱃지 표시 (실시간 폴링)

## 6. 제약 조건

| Phase | 범위 | 기간 | 비고 |

| 2. 인터뷰 UI | 채팅 인터페이스, 진행률, 중간 저장, PRD 생성 | 4일 | 비개발자 UX 검증 포함 |
| 3. 검토 UI | 피드백 뷰어, 스코어카드, PRD 편집기, 버전 관리 | 3일 | |
| 4. 연동 + QA | 아이디어/제안 연동, UAT, 버그 수정, 배포 | 3일 | 팀 UAT 1회 포함 |
| **합계** | | **14일 (약 3주)** | 버퍼 2일 포함 |

### 6.2 기술 스택
- Remix v2 + D1 + Drizzle + Tailwind + @axis-ds (기존 스택)
- AI 호출: Edge Runtime `fetch` (기존 `api.ideas.$id.analyze.ts` SSE 패턴 재사용)

### 6.3 기술 리스크 + 운영 장애 대응

| 리스크 | 대응 | Fallback |
|--------|------|----------|
| AI 병렬 호출 30초 타임아웃 | `Promise.allSettled` + 개별 25초 타임아웃 | 부분 성공 허용 (1개만 성공해도 피드백 표시) |
| AI API 장애/속도 저하 | 4단계 fallback: OpenAI → Gemini → DeepSeek → 수동 검토 안내 | 기존 radar-worker 패턴 재사용 |
| D1 동시 쓰기 경합 | 검토 결과 JSON 컬럼 1개 저장 | 개별 INSERT 불필요 |
| SSE 연결 끊김 | 폴링 fallback `/api/prd-studio/review/:id/status` | 30초 간격 |
| API 비용 초과 | 월간 사용량 추적 + 임계치(80%) 알림 | cost 모듈 기존 budget_policies 활용 |

### 6.4 테스트/QA 계획

| 단계 | 내용 | 담당 |
|------|------|------|
| 단위 테스트 | 인터뷰 상태 머신, PRD 생성, 스코어카드 계산 | Sinclair |
| 통합 테스트 | AI API 호출 → 피드백 파싱 → DB 저장 | Sinclair |
| UAT | 비개발자 팀원 1명이 실제 PRD 1건 작성 (Phase 4) | BD팀원 |
| 예외 처리 | API 실패, 타임아웃, 빈 응답, 잘린 응답 | Sinclair |

### 6.5 모니터링/로깅

- **이벤트 로그**: PRD 생성/검토 시작/완료/실패를 `cron_logs` 테이블에 기록
- **비용 추적**: 검토 API 호출 시 토큰 사용량을 `usage_events` 테이블에 기록 (기존 cost 모듈 활용)
- **알림**: 월간 비용 80% 임계치 도달 시 관리자 알림 (기존 alert 인프라)
- **에러 대시보드**: 기존 `/admin/monitoring`에 PRD Studio 섹션 추가 (API 성공률, 평균 응답 시간)

### 6.6 인력/예산
- 투입: 1명 (Sinclair) — 장애 시 Escalation: BD팀 리더에게 서비스 일시 중단 알림
- AI API: PRD당 ~$0.05 (review 2모델), ~$0.20 (sixhats 20턴), 월 $30 미만

### 6.7 민감 정보 정책
- **전송 전 고지**: 검토 버튼에 "PRD 내용이 외부 AI(OpenAI, Google)로 전송됩니다" 표시
- **작성 지침**: "개인정보, 인사정보, 미공개 재무정보 포함 금지" 인터뷰 시작 화면에 명시
- **기술 보호**: API 키 서버사이드 전용 / 원본 PRD 별도 로깅 안 함
- **인증**: 기존 Google OAuth + 세션 준수
