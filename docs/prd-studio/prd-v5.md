# PRD Studio PRD

**버전:** v5 (최종 후보)
**날짜:** 2026-03-11
**작성자:** AX BD팀
**상태:** 🔄 검토 중

---

## 1. 요약

**한 줄 정의:**
Discovery-X 앱 내에서 대화형 인터뷰 → PRD 자동 생성 → AI 다중 검토 → 피드백 반영 → 착수 판단까지 수행하는 웹 UI 기능.

**배경:** CLI 전용 인터뷰 스킬의 접근성 한계 + Discovery-X 워크플로우 분리 해소.

**목표:** 비개발자 포함 팀 전체 웹 PRD 기획 + 아이디어→PRD→제안 원스톱 + CLI 수준 검토 품질.

---

## 2. 문제 정의

**As-Is:** CLI 전용(비개발자 4명 접근 불가), 로컬 스크립트 의존, 도구 분산, 결과 로컬 파일.
**To-Be:** 웹 UI 원스톱 (인터뷰→PRD→검토→피드백→착수), 아이디어/제안 양방향 연동.
**시급성:** 운영 실험 중 비개발자 기획 참여 수단 부재.

## 3. 사용자

### 3.1 주 사용자
| 구분 | 주요 니즈 |
|------|-----------|
| BD팀 비개발자 (4명) | 가이드 기반 웹 PRD 작성, 기술 지식 불필요 |
| BD팀 개발자 (1명) | 아이디어→PRD→제안 앱 내 완결 |

### 3.2 핵심 사용자 여정

**여정 A (비개발자):** 아이디어 상세 → "PRD 작성" → 온보딩(최초) → 인터뷰(5파트, 예시 제공) → PRD 생성 → 검토(외부 전송 고지) → 피드백+스코어카드 → 수정/재검토 → 착수.

**여정 B (독립):** Lab > PRD Studio → 새 PRD → 인터뷰 → (A와 동일).

### 3.3 비개발자 UX 설계
- **온보딩**: 최초 3단계 스포트라이트 (기존 패턴 재사용)
- **예시 답변**: 파트별 접힘, 진행률 5파트 프로그레스 바
- **중간 저장**: DB → 재진입 이어하기
- **접근성**: 키보드 내비게이션, 충분한 색상 대비 (기존 @axis-ds WCAG AA 준수)

### 3.4 이탈 분석

| 이벤트 | 시점 | 기록 |
|--------|------|------|
| `interview_start` | 인터뷰 시작 | userId, prdId, source |
| `interview_part_complete` | 파트 완료 | partNumber, elapsed |
| `interview_abandon` | 이탈 | lastPart, totalElapsed |
| `prd_generated` | PRD 생성 | version, wordCount |
| `review_start/complete` | 검토 시작/완료 | models, round, score |
| `prd_finalized` | 착수 확정 | finalScore, rating |

## 5. 성공 기준

### 5.1 KPI
| 지표 | 현재 | 목표 | 측정 |
|------|------|------|------|
| 비개발자 PRD 작성 | 0건 | 팀원당 1+/분기 | DB |
| 인터뷰 완주율 | - | 80%+ | abandon 이벤트 |
| 검토 완주율 | 50% | 90%+ | review_complete |
| 인터뷰~착수 소요 | 25분+ | 15분 이내 | 타임스탬프 |
| 만족도 | - | 긍정 70%+ | 착수 시 평점+코멘트 |

### 5.2 MVP
- [ ] 5파트 인터뷰 + 중간 저장 + 온보딩
- [ ] PRD 생성 + 편집
- [ ] 2+ AI 검토 + 피드백 구조화
- [ ] 스코어카드 + 이탈 로깅

### 5.3 AI 검토 품질 검증 (PoC)
Phase 1에서 동일 PRD를 CLI(`review-api.mjs`)와 웹 API로 각각 검토하여 비교:
- verdict 일치율 목표: 100%
- 피드백 항목 수 차이: ±20% 이내
- 불일치 시 웹 API 프롬프트 조정 후 재검증

### 5.4 실패 + 개선 루프
| 조건 | 개선 | 중단 |
|------|------|------|
| 3개월 비개발자 미사용 | 1:1 인터뷰 → UX 개선 | 개선 후 1개월 미사용 |
| API $30+/월 | mini 다운그레이드 | 다운그레이드 후에도 초과 |
| 완주율 50% 미만 | 이탈 분석 → UX 개선 | 2회 개선 후 미달 |
| PoC 품질 불일치 | 프롬프트 튜닝 | 3회 튜닝 후 불일치 |

| 4. 연동 + QA | 아이디어/제안 연동, **UAT 팀원 3명+**, 버그 수정, 배포 | 4일 |
| **합계** | | **18일 (~4주)** |

> 이전 일정(15일)에서 PoC 품질 비교(+1일), UX 검증(+1일), UAT 확대(+1일) 반영.

### 6.2 배포/롤백 계획
- **배포**: 기존 CI/CD (GitHub Actions → Cloudflare Pages), master push 자동 배포
- **Feature Flag**: `PRD_STUDIO_ENABLED` — 초기 false, Phase 4 UAT 후 true 전환
- **롤백**: Feature Flag false로 즉시 비활성화 (코드 롤백 불필요)
- **스테이징**: `/ax-03-deploy --preview`로 프리뷰 배포 후 UAT 진행

### 6.3 기술 리스크 + 장애 대응

| 리스크 | 대응 | Fallback |
|--------|------|----------|
| AI 30초 타임아웃 | `Promise.allSettled` + 25초 | 부분 성공 허용 |
| AI API 장애 | 4단계 fallback | 수동 검토 안내 |
| SSE 끊김 | 폴링 30초 | status API |
| 비용 초과 | 월간 추적 + 80% 알림 | budget_policies |

### 6.4 테스트/QA

| 단계 | 내용 | 시점 |
|------|------|------|
| 단위 테스트 | 인터뷰 상태, PRD 생성, 스코어카드 | Phase 1~3 |
| PoC 품질 검증 | CLI vs 웹 동일 PRD 검토 비교 | Phase 1 |
| UX 검증 | 비개발자 1명 인터뷰 플로우 테스트 | Phase 2 완료 시 |
| UAT | 팀원 3명+ 실제 PRD 작성 | Phase 4 |
| 예외 | API 실패, 빈 응답, 세션 만료 | Phase 1~3 |

### 6.5 CLI ↔ 웹 호환성
- 동일 마크다운 템플릿 / CLI는 개발자 전용 유지 / 웹은 팀 전체
- 독립 채널 병행 (데이터 이관 불필요)

### 6.6 민감 정보 + 보안
- 전송 전 확인 다이얼로그 / 작성 금지 지침 명시
- API 키 서버사이드 전용 / 원본 별도 로깅 안 함
- 삭제 시 cascade / 유출 시 관리자 알림 + 기록 삭제
- 기존 인증(OAuth+세션) + requireUser() 가드

### 6.7 인력/예산
- 1명 (Sinclair) — Escalation: BD팀 리더 (장애 시 서비스 일시 중단)
- API: PRD당 ~$0.05, sixhats ~$0.20, 월 $30 미만
